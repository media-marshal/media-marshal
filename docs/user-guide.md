# Media Marshal 用户手册

> 本文档是 Media Marshal 的使用指导初版，面向希望把下载目录中的电影、剧集自动整理到 Emby / Jellyfin / Plex 媒体库的用户。
> 部署相关内容请参考 `docs/deployment.md`，常见问题会逐步沉淀到 `docs/faq.md`。

## 1. 项目能做什么

Media Marshal 是一个媒体文件自动整理工具，主要帮助你完成：

- 从指定源目录发现视频文件
- 使用 guessit 解析文件名中的标题、年份、季集号、分辨率等信息
- 使用 TMDB 匹配电影 / 剧集元数据
- 对低置信度或无法确定的任务进入待确认队列
- 按路径模板整理文件名和目录结构
- 移动、复制、硬链接或符号链接到目标媒体库目录
- 可选处理字幕、NFO、封面等附属文件
- 通过 Web 页面查看任务、确认候选、处理异常

适用人群：

- 使用 Emby / Jellyfin / Plex 管理本地媒体库的用户
- 希望把下载目录、临时整理目录、NAS 目录中的文件自动归档的用户
- 希望保留人工确认能力，避免识别错误直接污染媒体库的用户

当前不做：

- 不下载内容
- 不集成 PT 站、下载器或播放器
- 不提供用户认证，建议仅在本机或可信内网使用
- 不替代 Emby / Jellyfin / Plex 的刮削和播放能力

## 2. 基本概念

### 源目录

源目录是 Media Marshal 观察和扫描的目录，通常是下载完成目录、手动整理临时目录或 NAS 上的待处理目录。

### 目标目录

目标目录是整理后的媒体库根目录，例如电影库、剧集库或 Jellyfin / Emby / Plex 已配置的媒体目录。

### 路径规则

路径规则定义“从哪里发现文件、整理到哪里、按什么方式整理”。一条规则通常包含：

- 规则名称
- 源目录
- 目标目录
- 媒体类型
- 路径模板
- 文件操作方式
- 文件发现模式
- 附属文件、NFO、忽略规则等选项

### 媒体类型

媒体类型有三种：

- `MOVIE`：只按电影处理
- `TV_SHOW`：只按剧集处理
- `AUTO`：自动识别电影或剧集

如果同一个源目录里可能同时出现电影和剧集，可以使用 `AUTO`，并分别配置电影模板和剧集模板。

### 待确认队列

当系统没有找到候选，或候选置信度不足时，任务会进入待确认队列。你可以在 Queue 页面查看候选、手动搜索 TMDB、选择正确条目并继续整理。

### 附属文件

附属文件是和视频同目录、需要跟随主视频处理的文件，例如：

- 字幕：`.srt`、`.ass`、`.ssa`、`.sub`、`.idx`
- 用户自带 NFO：`.nfo`
- 同名封面：`.jpg`、`.png`、`.webp`
- 通用封面：`poster.jpg`、`folder.jpg`、`cover.jpg`

### 忽略文件规则

忽略规则用于静默跳过临时文件、系统文件、样片目录等。被忽略的文件不会生成任务，也不会出现在任务列表中。

## 3. 快速上手

### 1. 启动服务

按部署指南启动后，打开 Web UI。默认生产前端端口为 `3000`，后端服务端口为 `8080`。

### 2. 配置 TMDB API Key

进入“设置 → 系统设置”，填写 TMDB API Key。没有 API Key 时，系统无法完成元数据搜索和自动匹配。

建议同时确认：

- TMDB 查询语言，例如 `zh-CN` 或 `en-US`
- 低置信度阈值
- 邮件通知是否开启

### 3. 创建路径规则

进入“设置 → 路径设置”，新增一条路径规则。

最小可用配置：

- 规则名称：例如“电影下载目录”
- 源目录：待整理文件所在目录
- 目标目录：媒体库目录
- 媒体类型：电影、剧集或自动
- 文件操作方式：移动、复制、硬链接或符号链接
- 发现模式：建议优先使用混合模式

### 4. 手动扫描或等待自动发现

保存规则后，后续新文件会按发现模式自动进入处理流程。你也可以在页面上触发手动全量扫描，用于处理已经存在的文件。

### 5. 查看结果

在 Dashboard 查看最近任务，在 Queue 页面处理需要人工确认的任务。如果任务失败，可在任务详情列查看原因。

## 4. 路径规则配置

### 规则名称

名称只用于页面展示，建议用容易理解的名字，例如：

- 电影下载
- 剧集下载
- NAS 临时目录
- 4K 电影硬链接

### 源目录和目标目录

源目录和目标目录都应使用容器内可见路径。如果你使用 Docker 部署，请注意宿主机路径和容器内挂载路径不是同一个概念。

示例：

```text
源目录：/downloads
目标目录：/media/Movies
```

### 路径模板

路径模板决定目标目录下的相对路径和文件名。

电影默认模板示例：

```text
{title} ({year})/{title} ({year})[[ - {resolution}]]{ext}
```

剧集默认模板示例：

```text
{title} ({year})/S{season:02d}/{title} ({year}) - S{season:02d}E{episode:02d}[[ - {resolution}]]{ext}
```

常用变量：

- `{title}`：标题
- `{original_title}`：原始标题
- `{year}`：年份
- `{season:02d}`：两位季号
- `{episode:02d}`：两位集号
- `{resolution}`：分辨率
- `{ext}`：原扩展名，包含点号

可选片段语法：

```text
[[ - {resolution}]]
```

当片段内变量有值时保留整个片段；任意变量缺失时移除整个片段。这样可以避免分辨率缺失时生成 ` - {resolution}` 这样的文件名。

### 模板帮助和预览

路径设置页提供模板变量帮助和自定义模板预览。保存前建议先预览路径样例，确认目录层级和文件名是否符合媒体服务器习惯。

## 5. 文件发现与扫描

Media Marshal 支持多种发现方式。

### 实时监听

实时监听依赖系统文件事件，适合本地磁盘。优点是响应快，缺点是在 NAS、Docker volume、网络挂载场景中可能漏事件。

### 定时扫描

定时扫描按规则周期扫描源目录，不依赖文件事件。适合 NAS、网络挂载、Docker volume 等文件事件不稳定的场景。

### 混合模式

混合模式同时使用实时监听和定时扫描，是当前推荐模式。它兼顾实时性和可靠性。

### 手动全量扫描

如果你刚创建规则，或怀疑自动发现漏掉了文件，可以手动触发全量扫描。手动扫描也会遵守忽略规则、数据库查重、文件分类和待确认流程。

### 新目录补扫

当系统发现新目录时，会注册监听并立即对该目录做一次轻量扫描，避免外部工具快速创建目录和文件时漏掉文件。

## 6. 文件操作方式

文件操作方式决定源文件和目标文件之间的关系。

| 操作 | 行为 | 源文件是否保留 |
|---|---|---|
| 移动 | 将源文件移动到目标路径 | 否 |
| 复制 | 复制一份到目标路径 | 是 |
| 硬链接 | 在目标路径创建指向同一份文件内容的硬链接 | 是 |
| 符号链接 | 在目标路径创建指向源路径的符号链接 | 是 |

### 保存前校验

创建或编辑规则时，系统会进行保存前校验。例如：

- 源目录是否存在
- 目标目录是否可写
- 硬链接是否可创建
- 符号链接是否有权限创建

校验失败时不会保存规则，需要根据提示修正路径、权限或操作方式。

### 目标路径冲突

如果目标路径已经存在，系统不会覆盖，也不会自动改名。任务会失败，并记录类似“目标文件已存在，文件冲突”的原因。

这是为了避免静默覆盖已经整理好的媒体文件。

### 硬链接注意事项

硬链接通常要求源目录和目标目录在同一文件系统或同一挂载设备上。跨磁盘、跨 NAS 共享或跨 Docker volume 时可能失败。

### 符号链接注意事项

符号链接依赖运行环境权限。源文件删除或移动后，符号链接会失效。

## 7. 附属文件与 NFO

路径规则中可以配置是否自动移动附属文件、是否自动生成 NFO。

### 自动移动附属文件

开启后，主视频整理成功时，同目录下的附属文件会跟随处理。

同名字幕、NFO、封面会按目标视频文件名重命名。通用封面如 `poster.jpg`、`folder.jpg`、`cover.jpg` 会移动到目标目录并保留原文件名。

### 用户自带 NFO

如果源目录中存在用户自带 NFO，系统会优先处理用户 NFO，不覆盖它。

### 自动生成 NFO

自动生成 NFO 默认关闭。开启后，只有在源目录没有用户自带 NFO 时，系统才会生成新的 NFO。

### 附属文件失败

附属文件处理失败不会回滚已经成功处理的主视频，但系统会记录日志。后续版本可能在任务详情中展示更详细提示。

## 8. 忽略文件规则

忽略规则用于减少任务噪音。匹配忽略规则的文件或目录会被静默跳过，不生成任务记录。

系统默认规则包括：

```text
.DS_Store
Thumbs.db
desktop.ini
*.part
*.tmp
*.crdownload
*.lock
~$*
.*
__MACOSX/
@eaDir/
```

规则语义类似 gitignore 的核心子集：

- `Thumbs.db`：匹配任意层级下的同名文件
- `*.tmp`：匹配对应扩展名
- `.*`：匹配隐藏文件或目录
- `sample/`：匹配任意层级下名为 `sample` 的目录及其内容

路径规则中可以编辑忽略规则：

- `null`：使用系统默认规则
- 空列表：不启用任何忽略规则
- 自定义列表：使用该规则自己的完整忽略规则

如果一个文件没有出现在任务列表中，可能是它命中了忽略规则。

## 9. 待确认队列

任务进入待确认队列的常见原因：

- 没有找到合适候选
- 候选置信度低于阈值
- 媒体类型或标题解析不确定

### 查看系统候选

每个待确认任务会展示系统候选列表。你可以查看标题、原始标题、年份、媒体类型、海报、简介和置信度。

### 全局手动搜索

Queue 页面提供全局手动搜索面板。它不是某个任务的局部搜索，而是用于搜索一个候选，再把候选关联到一个或多个任务。

支持两种操作路径：

1. 先搜索候选，再勾选任务并应用候选
2. 先勾选任务，再搜索候选并应用候选

搜索结果只是前端临时候选，不会立即写入数据库。只有点击确认后，候选才会按任务记录保存。

### 批量确认

批量确认只会提交当前页中“已选择任务，且已关联候选”的任务。已选择但未关联候选的任务不会提交。

典型剧集批量确认流程：

1. 在 Queue 页面选择当前页任务
2. 搜索正确剧集，例如 `长乐曲`
3. 选择正确 TMDB 候选
4. 应用候选到已选任务
5. 点击批量确认

确认后，成功提交的任务会继续整理；失败任务会保留在队列或任务列表中，并显示失败原因。

### 系统推荐快捷面板

如果当前页多个任务有相同系统推荐，页面会展示“一键选择”推荐分组。默认只展示前 3 条，可展开全部。这个功能适合一批任务都推荐为同一部剧或同一部电影的情况。

### 单条确认

单个任务也可以走同一套流程：只选择一个任务，应用候选后确认即可。

### 跳过任务

如果你不想处理某个待确认任务，可以选择跳过。跳过会把任务标记为 `SKIPPED`，不移动、不复制、不链接文件。

## 10. 仪表盘和任务管理

Dashboard 展示最近任务和任务状态概览。任务列表支持筛选、分页和删除记录。

### 任务状态

| 状态 | 含义 |
|---|---|
| `PENDING` | 等待处理 |
| `PROCESSING` | 正在处理 |
| `AWAITING_CONFIRMATION` | 等待人工确认 |
| `DONE` | 已完成 |
| `FAILED` | 处理失败 |
| `SKIPPED` | 已跳过 |

### 详情列

详情列根据任务状态展示不同内容：

- `FAILED`：展示失败原因 `errorMessage`
- `SKIPPED`：展示跳过原因 `skipReason`
- 其他状态：通常为空或展示简短状态说明

### 删除任务记录

删除任务记录只删除数据库中的任务和候选记录，不影响已经存在的媒体文件。批量删除前会有确认提示。

## 11. 常见场景示例

### 场景一：下载目录整理到电影库

建议配置：

- 源目录：下载完成目录
- 目标目录：电影库目录
- 媒体类型：`MOVIE`
- 文件操作：移动或硬链接
- 发现模式：混合模式
- 电影模板：使用默认模板即可

### 场景二：剧集批量整理

建议配置：

- 媒体类型：`TV_SHOW` 或 `AUTO`
- 剧集模板使用季目录，例如 `S{season:02d}`
- Queue 页面使用全局手动搜索和批量确认，减少逐集确认成本

### 场景三：NAS / Docker volume

建议配置：

- 发现模式：混合模式或定时扫描
- 扫描间隔不要设置过短
- 如果实时监听漏文件，优先使用手动扫描验证文件是否可见

### 场景四：使用硬链接节省空间

硬链接可以在目标目录创建另一个目录项，而不复制文件内容。适合希望源目录保留做种、目标目录供媒体库使用的场景。

注意：

- 源目录和目标目录通常需要位于同一文件系统
- 不支持时保存前校验会失败
- 不会自动 fallback 到复制

## 12. 常见问题

### 为什么没有自动识别到新文件？

可能原因：

- 规则未启用
- 源目录路径在容器内不可见
- 文件还在写入，尚未通过稳定性检测
- 文件命中忽略规则
- 实时监听在 NAS / Docker volume 上漏事件

可以尝试手动全量扫描。如果手动扫描能发现，建议使用混合模式或定时扫描。

### 为什么任务进入待确认？

通常是候选置信度不足，或没有找到明确候选。你可以在 Queue 页面使用系统候选、一键推荐或全局手动搜索完成确认。

### 为什么提示目标文件冲突？

目标路径已经存在。系统不会覆盖已有文件，也不会自动追加后缀。你需要手动处理目标文件，或调整模板 / 目标目录后重新处理。

### 为什么硬链接失败？

常见原因：

- 源目录和目标目录不在同一文件系统
- Docker volume 或 NAS 不支持硬链接
- 目录权限不足

可以改用复制、移动或调整挂载方式。

### 为什么有些文件显示为已跳过？

普通非视频文件不会进入媒体识别流程，会生成 `SKIPPED` 任务并展示跳过原因。系统噪音和临时文件如果命中忽略规则，则会静默跳过，不生成任务。

### 如何处理识别错误？

如果任务还在 Queue 中，使用全局手动搜索选择正确候选后确认。如果已经整理完成，需要手动修正文件，或删除记录后重新扫描源文件。

## 13. 已知限制

当前版本仍有一些限制：

- 没有用户认证，建议仅在本机或可信内网使用
- 元数据主要依赖 TMDB，TMDB 不可用时匹配能力会受影响
- SQLite 更适合单实例部署，不建议多个后端实例同时写同一个数据库
- Webhook 文件发现能力仍是后续版本待讨论事项
- 附属文件失败暂不在任务详情中完整展示
- 完整跨分页批量选择尚未实现，Queue 批量操作只作用于当前页

如果你遇到无法判断的问题，建议先查看任务详情、后端日志和 FAQ，再决定是否调整规则或手动处理文件。
